# TradeNotti

Trading journal platform. This milestone implements the full backend and
frontend for the **Today** page:

- **Daily insight** — generated from the trader's journal data (OpenAI when an
  API key is configured, otherwise a deterministic stub), cached once per day.
  No "See report" / "Dismiss" actions, by design.
- **Today's trades** — currently running (open) positions, pulled from the
  broker (MetaTrader 5 via MetaApi) with a mock fallback. Live-refreshed.
- **Trading rules** — read-only feed of the rules authored in the (upcoming)
  Rules feature.

## Stack

- Next.js 15 (App Router, React 19, TypeScript)
- PostgreSQL + Prisma ORM
- Tailwind CSS v4
- Pluggable broker layer (MetaApi.cloud adapter + mock)
- Pluggable AI insight layer (OpenAI + deterministic stub)

## Quick start

```bash
# 1. Install
npm install

# 2. Configure env
cp .env.example .env
#    Edit DATABASE_URL if needed. Leave OPENAI_API_KEY blank to use the stub.
#    Leave BROKER_PROVIDER=mock to use seeded open trades.

# 3. Database: run migrations + seed demo data (Edwin + FTMO 100k account)
npm run db:migrate
npm run db:seed

# 4. Run
npm run dev          # http://localhost:3000  (redirects to /today)
```

## Environment variables

| Variable             | Purpose                                                        |
| -------------------- | -------------------------------------------------------------- |
| `DATABASE_URL`       | Postgres connection string                                     |
| `OPENAI_API_KEY`     | If set, insights are generated by OpenAI; else the stub is used |
| `OPENAI_MODEL`       | OpenAI model (default `gpt-4o-mini`)                           |
| `BROKER_PROVIDER`    | `mock` (default) or `metaapi`                                  |
| `METAAPI_TOKEN`      | MetaApi.cloud auth token (live MT5)                            |
| `METAAPI_ACCOUNT_ID` | MetaApi account id (per-account override stored on `Account`)  |
| `METAAPI_REGION`     | MetaApi region host (default `new-york`)                       |
| `DEMO_USER_EMAIL`    | Identifies the demo user until auth is added                   |

## Architecture

```
  app/
    (app)/today/page.tsx     Today dashboard (server component)
    api/                     accounts · trades · rules · insights/today · sync
  components/                Sidebar, TopBar, AccountSwitcher, today/*
  lib/
    db.ts                    Prisma client singleton
    account.ts               Current-user / active-account resolution (auth seam)
    trades.ts                Open-trade reads
    rules.ts                 Rule reads (Rules feature writes them later)
    broker/                  BrokerProvider interface + MetaApi + Mock adapters

    ai/
      stats.ts               Journal stats aggregation
      insight.ts             generateDailyInsight (OpenAI + stub)
      daily-insight.ts       Once-per-day caching layer
prisma/
  schema.prisma              User, Account, Trade, Tag, Rule, DailyInsight
  seed.ts                    Demo data
```

### Going live with MetaTrader 5

MT5 has no public REST API. To pull live open positions in the cloud, link a
real account through [MetaApi.cloud](https://metaapi.cloud), then set
`BROKER_PROVIDER=metaapi`, `METAAPI_TOKEN`, and the account's
`metaApiAccountId`. Until then the mock provider renders seeded positions.

Refresh open positions on demand:

```bash
npm run broker:sync           # all accounts
curl -X POST localhost:3000/api/sync   # active account
```

### Enabling real AI insights

Set `OPENAI_API_KEY`. The insight then comes from OpenAI using the same journal
stats the stub reads, so only the phrasing changes. Insights are cached one per
account per day; force a regenerate with `/api/insights/today?force=1`.
